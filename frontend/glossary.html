Admin (ผู้ดูแลระบบ)

อธิบายง่ายๆ: Admin คือคนที่ดูแลระบบทั้งหมด เหมือนเป็น "หัวหน้า" ที่มีสิทธิ์ทำได้ทุกอย่างในระบบ

ความหมาย: ผู้มีอำนาจสูงสุดในระบบ สามารถทำได้ทุกอย่าง รวมถึงกำหนดสิทธิ์ผู้ใช้ ตั้งค่ากฎเกณฑ์ และ Override ทุกอย่าง

สิทธิ์ที่ Admin มี:

  • เห็นทุกเมนูและข้อมูลในระบบ
  • แก้ไขข้อมูลได้ทุกอย่าง
  • ข้ามกฎหรือการเตือนได้ (Override)
  • ลบหรือแก้ไขบันทึกย้อนหลังได้
  • กำหนดสิทธิ์ให้ผู้ใช้อื่นได้
ตัวอย่าง: ถ้าคิวรถซ้ำกัน Admin สามารถ Override เพื่อให้ใช้รถได้ หรือถ้ามีคนลืมบันทึก Admin สามารถแก้ไขให้ได้

สิทธิ์การเข้าถึง (Permission)

อธิบายง่ายๆ: สิทธิ์คือ "กุญแจ" ที่ Admin มอบให้แต่ละคน เพื่อเปิดใช้ฟีเจอร์ต่างๆ ในระบบ

ความหมาย: สิทธิ์ที่ Admin กำหนดให้ผู้ใช้แต่ละคนในการเข้าถึงและทำงานในแต่ละโมดูล

ระดับสิทธิ์ (จากน้อยไปมาก):

  • ดูอย่างเดียว: เห็นข้อมูลได้ แต่แก้ไขไม่ได้
  • บันทึกใหม่: เพิ่มข้อมูลใหม่ได้ แต่แก้ไขข้อมูลเก่าไม่ได้
  • แก้ไข: แก้ไขข้อมูลที่มีอยู่แล้วได้
  • ลบ: ลบข้อมูลได้ (เฉพาะ Admin เท่านั้น)
ตัวอย่าง:
• ผู้ใช้ A มีสิทธิ์ "จัดคิวรถ" และ "ดูรายงาน" แต่ไม่มีสิทธิ์ "บันทึกน้ำมัน"
• ผู้ใช้ B มีสิทธิ์ "ดูรายงาน" อย่างเดียว ไม่สามารถแก้ไขอะไรได้
• Admin มีสิทธิ์ทุกอย่าง
💡 เคล็ดลับ: ถ้าคุณไม่เห็นเมนูหรือปุ่มบางอย่าง แสดงว่าคุณยังไม่มีสิทธิ์ในส่วนนั้น ต้องขอ Admin ให้เพิ่มสิทธิ์ให้

Audit Trail (บันทึกการเปลี่ยนแปลง)

อธิบายง่ายๆ: Audit Trail คือ "สมุดบันทึก" ที่ระบบจดบันทึกทุกครั้งที่มีคนแก้ไขข้อมูลสำคัญ เหมือนกล้องวงจรปิดที่บันทึกว่าใครทำอะไร เมื่อไหร่

ความหมาย: บันทึกอัตโนมัติของทุกการเปลี่ยนแปลงข้อมูลสำคัญในระบบ

ข้อมูลที่ระบบบันทึก:

  • ใครทำ: ชื่อผู้ใช้ที่แก้ไข
  • ทำเมื่อไหร่: วันและเวลาที่แก้ไข
  • แก้ไขอะไร: ข้อมูลส่วนไหนที่เปลี่ยน
  • ค่าเก่า: ข้อมูลเดิมเป็นอย่างไร
  • ค่าใหม่: ข้อมูลใหม่เป็นอย่างไร
ตัวอย่าง: ถ้ามีคนแก้ไขคิวรถจาก "รถ กข 1234" เป็น "รถ กข 5678" ระบบจะบันทึกว่า "คุณสมชาย แก้ไขเมื่อ 15 ม.ค. 2569 เวลา 10:30 น. เปลี่ยนจาก กข 1234 เป็น กข 5678"
💡 ข้อสำคัญ: Audit Trail ใช้เพื่อตรวจสอบย้อนหลังและความโปร่งใสเท่านั้น ไม่ใช้เพื่อลงโทษ เพราะทุกคนอาจทำผิดพลาดได้

PDPA (Personal Data Protection Act)

อธิบายง่ายๆ: PDPA คือกฎหมายที่ปกป้องข้อมูลส่วนตัวของทุกคน ระบบจะเก็บข้อมูลเท่าที่จำเป็น และไม่ให้คนอื่นเห็นข้อมูลส่วนตัวของคุณโดยไม่ได้รับอนุญาต

ความหมาย: พ.ร.บ.คุ้มครองข้อมูลส่วนบุคคล พ.ศ. 2562 (กฎหมายที่ปกป้องข้อมูลส่วนตัว)

ในระบบ PPK DriveHub:

  • ใช้ข้อมูลเท่าที่จำเป็น: เก็บเฉพาะข้อมูลที่ต้องใช้จริงๆ
  • ควบคุมการเข้าถึง: มีสิทธิ์เท่านั้นถึงจะเห็นข้อมูลได้
  • บันทึกการเข้าดู: บันทึกว่าใครดูข้อมูลส่วนตัวของคุณเมื่อไหร่
ตัวอย่าง: ข้อมูลส่วนตัวเช่น เบอร์โทรศัพท์ ที่อยู่ จะมีแค่ Admin และคนที่เกี่ยวข้องเท่านั้นที่เห็นได้ คนอื่นจะเห็นแค่ชื่อเท่านั้น

🚗 การจัดการคิวรถ

คิวรถ (Queue)

อธิบายง่ายๆ: คิวรถคือการ "จอง" รถและคนขับไว้ล่วงหน้า เหมือนจองโต๊ะอาหารที่ร้านอาหาร แต่จองรถและคนขับแทน

ความหมาย: การจองรถและคนขับสำหรับงานในวันและเวลาที่กำหนด

ข้อมูลที่ต้องกรอกเมื่อจองคิว:

  • วันที่: วันที่จะใช้รถ
  • เวลาไป: เวลาที่ออกจากโรงเรียน
  • เวลากลับ: เวลาที่คาดว่าจะกลับ
  • รถ: รถคันไหนที่ต้องการใช้
  • คนขับ: ต้องการให้ใครขับ (หรือให้ระบบเลือกให้)
  • ผู้ขอใช้: ใครเป็นคนขอใช้รถ
  • สถานที่ไป: ไปที่ไหน
  • จำนวนผู้โดยสาร: มีกี่คนนั่งไปด้วย
ตัวอย่าง: คุณต้องการไปประชุมที่จังหวัดเชียงใหม่ในวันที่ 20 ม.ค. เวลา 8:00 น. กลับเวลา 17:00 น. คุณก็จองคิวรถไว้ล่วงหน้า ระบบจะจัดรถและคนขับให้

Freeze คิว

อธิบายง่ายๆ: Freeze คิวคือการ "ล็อก" คิวไว้ไม่ให้แก้ไขเมื่อใกล้เวลาออกเดินทาง เหมือนการปิดประตูห้องสอบก่อนเริ่มสอบ เพื่อป้องกันการแก้ไขผิดพลาดในนาทีสุดท้าย

ความหมาย: การล็อกคิวเมื่อใกล้เวลาออก (เช่น 30 นาทีก่อน) เพื่อป้องกันการแก้ไขผิดพลาด

ผลที่เกิดขึ้น:

  • คิวที่ Freeze แล้วไม่สามารถแก้ไขได้
  • ไม่สามารถยกเลิกคิวได้
  • ยกเว้น Admin เท่านั้นที่สามารถแก้ไขได้
ตัวอย่าง: ถ้าคุณจองคิวไว้เวลา 8:00 น. เมื่อถึงเวลา 7:30 น. (30 นาทีก่อน) ระบบจะ Freeze คิวนี้ไว้ คุณจะแก้ไขหรือยกเลิกไม่ได้แล้ว แต่ถ้าจำเป็นจริงๆ Admin สามารถแก้ไขให้ได้
💡 เคล็ดลับ: ถ้าคุณต้องการแก้ไขคิว ควรแก้ไขก่อนเวลา 30 นาทีก่อนออกเดินทาง หรือติดต่อ Admin ถ้าจำเป็นจริงๆ

Override (ยกเว้นกฎ)

อธิบายง่ายๆ: Override คือการ "ข้าม" กฎหรือการเตือนของระบบเมื่อมีเหตุผลจำเป็นจริงๆ เหมือนการขออนุญาตข้ามไฟแดงในกรณีฉุกเฉิน

ความหมาย: การข้ามกฎหรือการเตือนของระบบเมื่อมีเหตุผลจำเป็น

เงื่อนไขในการ Override:

  • ต้องมีสิทธิ์ Override (Admin หรือผู้ที่ได้รับอนุญาต)
  • ต้องกรอกเหตุผลว่าทำไมต้อง Override
  • ระบบจะบันทึกว่าใคร Override อะไร เมื่อไหร่
ตัวอย่าง:
• รถซ้ำกับคิวอื่น แต่เป็นงานด่วนจำเป็น → Override ได้
• คนขับล้า (Fatigue) แต่เป็นงานสำคัญมาก → Override ได้ (แต่ต้องระวังความปลอดภัย)
• จำนวนผู้โดยสารเกินที่นั่ง แต่เป็นกรณีพิเศษ → Override ได้
⚠️ คำเตือน: Override ควรใช้เฉพาะเมื่อจำเป็นจริงๆ เท่านั้น เพราะอาจส่งผลต่อความปลอดภัยหรือประสิทธิภาพการทำงาน

Timeline View (มุมมองไทม์ไลน์)

ความหมาย: การแสดงคิวรถและคิวคนขับในรูปแบบเส้นเวลา

ประโยชน์: เห็นภาพรวมการใช้งานรถและคนขับในแต่ละวัน/สัปดาห์

สถานะคิว

ความหมาย: สถานะของคิวในแต่ละช่วงเวลา

สถานะ: ร่าง, ยืนยัน, freeze, ใช้งาน, เสร็จ, ยกเลิก

👨‍✈️ พนักงานขับรถ

พนักงานขับรถหลัก

ความหมาย: พนักงานขับรถประจำที่มีบทบาทชัดเจน

ประเภท: ภารกิจภายนอก, ภารกิจภายใน, ภารกิจผู้บริหาร

พนักงานขับรถสำรอง

ความหมาย: พนักงานขับรถที่ใช้เมื่อพนักงานขับรถหลักไม่ว่างหรือลา

ความล้า (Fatigue)

อธิบายง่ายๆ: ความล้าคือสถานะที่คนขับขับรถมากเกินไปในวันก่อน เหมือนคนที่ทำงานหนักมากเมื่อวาน วันนี้ควรพักผ่อนเพื่อความปลอดภัย

ความหมาย: สถานะที่คนขับขับรถเกินระยะทางที่กำหนด (เช่น 400 กม.) ในวันก่อน

ผลที่เกิดขึ้น:

  • ระบบจะเตือนว่าคนขับล้า
  • อาจบังคับให้พัก (ตามที่ Admin ตั้งค่า)
  • อาจไม่ให้จัดคิวใหม่จนกว่าจะพักผ่อนเพียงพอ
ตัวอย่าง: ถ้าคนขับขับรถไป 450 กม. เมื่อวาน วันนี้ระบบจะเตือนว่า "คนขับล้า" และอาจไม่ให้จัดคิวใหม่จนกว่าจะพักผ่อนเพียงพอ
💡 ข้อสำคัญ: ความล้าเป็นเรื่องสำคัญมาก เพราะอาจทำให้เกิดอุบัติเหตุได้ ควรให้คนขับพักผ่อนให้เพียงพอก่อนขับรถอีกครั้ง

Standby (สแตนด์บาย)

ความหมาย: สถานะที่คนขับพร้อมทำงานแต่ยังไม่มีคิว

คะแนนวินัยการใช้งานรถ

ความหมาย: คะแนนที่ประเมินจากการบันทึกการใช้งานรถอย่างสม่ำเสมอ

ใช้เพื่อ: การพัฒนา ไม่ใช่การลงโทษ

⛽ การจัดการน้ำมัน

บันทึกน้ำมัน (Fuel Log)

ความหมาย: บันทึกการเติมน้ำมันของรถแต่ละคัน

ข้อมูล: ประเภทน้ำมัน, จำนวนลิตร, ราคาต่อลิตร, ราคารวม, เลขไมล์, รูปสลิป

อัตราสิ้นเปลือง (Fuel Consumption)

อธิบายง่ายๆ: อัตราสิ้นเปลืองคือ "รถกินน้ำมันมากแค่ไหน" วัดเป็น "กี่กิโลเมตรต่อลิตร" ยิ่งตัวเลขมากยิ่งดี เพราะหมายความว่ารถวิ่งได้ไกลด้วยน้ำมันน้อย

ความหมาย: จำนวนกิโลเมตรต่อลิตร (กม./ลิตร) ที่รถใช้

การคำนวณ: ระยะทาง (กม.) ÷ ปริมาณน้ำมัน (ลิตร)

ตัวอย่าง:
• รถวิ่งได้ 100 กิโลเมตร ใช้น้ำมัน 10 ลิตร → อัตราสิ้นเปลือง = 100 ÷ 10 = 10 กม./ลิตร
• รถวิ่งได้ 150 กิโลเมตร ใช้น้ำมัน 10 ลิตร → อัตราสิ้นเปลือง = 150 ÷ 10 = 15 กม./ลิตร (ดีกว่า)
• รถวิ่งได้ 80 กิโลเมตร ใช้น้ำมัน 10 ลิตร → อัตราสิ้นเปลือง = 80 ÷ 10 = 8 กม./ลิตร (แย่กว่า)
💡 เคล็ดลับ: ถ้าอัตราสิ้นเปลืองต่ำกว่าปกติ (เช่น จาก 12 กม./ลิตร เหลือ 8 กม./ลิตร) อาจหมายความว่ารถมีปัญหา ควรตรวจสอบ

Threshold แจ้งเตือนน้ำมันผิดปกติ

ความหมาย: ค่าที่ Admin ตั้งไว้เพื่อแจ้งเตือนเมื่อรถใช้น้ำมันผิดปกติ

ตัวอย่าง: ถ้าใช้น้ำมันมากกว่าค่าเฉลี่ย 20% ระบบจะแจ้งเตือน

🛠️ การซ่อมบำรุง

ซ่อมตามแผน (Scheduled Repair)

ความหมาย: การซ่อมที่วางแผนไว้ล่วงหน้าตามระยะทางหรือเวลา

ตัวอย่าง: เปลี่ยนน้ำมันเครื่องทุก 10,000 กม. หรือ 6 เดือน

ซ่อมฉุกเฉิน (Emergency Repair)

ความหมาย: การซ่อมที่เกิดขึ้นทันทีเนื่องจากความเสียหายที่ไม่คาดคิด

สถานะซ่อม

ความหมาย: สถานะของรถในกระบวนการซ่อม

สถานะ: รอซ่อม, กำลังซ่อม, ซ่อมเสร็จ, เลื่อน

คะแนนความพร้อมใช้งานรถ

ความหมาย: คะแนนที่ประเมินจากความพร้อมใช้งานของรถ

ปัจจัย: อัตราการใช้งาน, ความถี่ซ่อม, ค่าใช้จ่าย, ความเสื่อมสภาพ

การตรวจสภาพรถ (Daily Check)

ความหมาย: การตรวจสอบสภาพรถพื้นฐานก่อนใช้งานทุกวัน

รายการ: ระดับของเหลว, แรงดันลมยาง, ท่อยางรั่วซึม, เครื่องยนต์/เกียร์/เฟืองท้าย, ระบบไฟ

📊 การบันทึกและการรายงาน

บันทึกการใช้รถ (Usage Record)

ความหมาย: บันทึกการใช้งานรถแต่ละครั้ง

ประเภท: ก่อนออกเดินทาง, กลับจากเดินทาง

ข้อมูล: วันเวลา, คนขับ, ผู้ขอใช้, จุดหมายปลายทาง, เลขไมล์

Auto-Recovery (บันทึกอัตโนมัติ)

อธิบายง่ายๆ: Auto-Recovery คือ "ผู้ช่วยอัตโนมัติ" ที่จะบันทึกให้คุณอัตโนมัติเมื่อคุณลืมบันทึก เหมือนเพื่อนที่ช่วยจดบันทึกให้เมื่อคุณลืม

ความหมาย: ระบบบันทึกให้อัตโนมัติเมื่อลืมบันทึกก่อนออกหรือหลังกลับ

ผลที่เกิดขึ้น:

  • ระบบจะบันทึกให้อัตโนมัติเมื่อคุณลืม
  • บันทึกจะถูกทำเครื่องหมายว่า "ระบบบันทึกแทน"
  • ระบบจะเก็บสถิติว่าคุณลืมบันทึกกี่ครั้ง
ตัวอย่าง: คุณขับรถออกไปแล้วลืมบันทึก ระบบจะบันทึกให้อัตโนมัติโดยใช้ข้อมูลจากคิวรถที่คุณจองไว้ แต่จะทำเครื่องหมายว่า "ระบบบันทึกแทน" เพื่อให้รู้ว่าคุณลืมบันทึก
💡 เคล็ดลับ: แม้ว่าระบบจะบันทึกให้อัตโนมัติ แต่ควรพยายามบันทึกเองทุกครั้งเพื่อความถูกต้องและครบถ้วน

QR Code (คิวอาร์โค้ด)

อธิบายง่ายๆ: QR Code คือ "รหัสสี่เหลี่ยม" ที่ติดอยู่บนรถแต่ละคัน เมื่อสแกนด้วยมือถือจะเปิดหน้าจอสำหรับบันทึกข้อมูลทันที เหมือนบาร์โค้ดแต่ใช้มือถือสแกนได้

ความหมาย: รหัส QR ที่ติดอยู่บนรถแต่ละคัน สำหรับสแกนเพื่อบันทึกข้อมูล

วิธีใช้:

  1. เปิดกล้องมือถือหรือแอปสแกน QR Code
  2. สแกน QR Code ที่ติดอยู่บนรถ
  3. ระบบจะเปิดหน้าจอสำหรับบันทึกข้อมูลทันที
  4. กรอกข้อมูลและบันทึก

ใช้สำหรับ:

  • บันทึกการใช้รถ: บันทึกก่อนออกและหลังกลับ
  • บันทึกน้ำมัน: บันทึกการเติมน้ำมัน
  • ตรวจสภาพรถ: บันทึกการตรวจสภาพรถประจำวัน
ตัวอย่าง: เมื่อคุณจะขับรถออก คุณสแกน QR Code ที่ติดอยู่บนรถ ระบบจะเปิดหน้าจอให้คุณกรอกข้อมูล เช่น เลขไมล์ จุดหมายปลายทาง แล้วบันทึก
💡 เคล็ดลับ: QR Code ทำให้บันทึกข้อมูลได้ง่ายและรวดเร็ว ไม่ต้องพิมพ์เลขทะเบียนรถเอง ระบบจะรู้ทันทีว่าคุณใช้รถคันไหน

Dashboard (หน้าแรก)

อธิบายง่ายๆ: Dashboard คือ "หน้าจอหลัก" ที่แสดงภาพรวมของวันนี้ เหมือนหน้าปัดรถยนต์ที่แสดงข้อมูลสำคัญทั้งหมดในที่เดียว

ความหมาย: หน้าหลักที่แสดงสถานะวันนี้เท่านั้น

ข้อมูลที่แสดง:

  • คนขับ: ใครกำลังทำงาน ใครว่าง
  • รถ: รถคันไหนออกงาน รถคันไหนว่าง รถคันไหนซ่อม
  • คิววันนี้: มีคิวอะไรบ้างในวันนี้
  • ปฏิทิน: แสดงคิวในรูปแบบปฏิทิน
ตัวอย่าง: เมื่อคุณเปิด Dashboard คุณจะเห็นทันทีว่า "วันนี้มีคิว 5 คิว คนขับ 3 คนกำลังทำงาน รถ 2 คันออกงาน รถ 1 คันซ่อม"
💡 เคล็ดลับ: Dashboard เป็นหน้าหลักที่ควรเปิดดูทุกวันเพื่อดูภาพรวมของวันนี้

รายงานประเมินผล (Evaluation Report)

ความหมาย: รายงานที่ใช้ประเมินผลการทำงานในช่วงเวลาที่กำหนด

ใช้เพื่อ: ตัดสินใจ, ทำรายงานราชการ, ประเมินประสิทธิภาพ

ช่วงเวลา: รายเดือน, รายไตรมาส, รายปีงบประมาณ, Custom Range

KPI (Key Performance Indicator)

อธิบายง่ายๆ: KPI คือ "คะแนน" ที่บอกว่ารถหรือคนขับทำงานได้ดีแค่ไหน เหมือนคะแนนสอบที่บอกว่าคุณทำได้ดีแค่ไหน

ความหมาย: ตัวชี้วัดประสิทธิภาพหลัก (คะแนนที่บอกว่าทำงานได้ดีแค่ไหน)

ตัวอย่าง KPI:

  • อัตราการใช้งานรถ: รถถูกใช้งานบ่อยแค่ไหน
  • ค่าเฉลี่ยระยะทางต่อวัน: รถวิ่งเฉลี่ยกี่กิโลเมตรต่อวัน
  • ค่าใช้จ่ายต่อกิโลเมตร: ใช้เงินเฉลี่ยกี่บาทต่อกิโลเมตร
  • อัตราสิ้นเปลือง: รถกินน้ำมันมากแค่ไหน

ระดับคะแนน:

  • 80-100 คะแนน: ดีมาก (สีเขียว) ✅
  • 60-79 คะแนน: ปกติ (สีเหลือง) ⚠️
  • น้อยกว่า 60 คะแนน: ควรพัฒนา (สีแดง) ❌
ตัวอย่าง:
• รถคัน A มี KPI = 85 คะแนน → ดีมาก (ใช้งานดี ค่าใช้จ่ายต่ำ)
• รถคัน B มี KPI = 65 คะแนน → ปกติ (ใช้งานได้ แต่ควรปรับปรุง)
• รถคัน C มี KPI = 45 คะแนน → ควรพัฒนา (มีปัญหาต้องแก้ไข)
💡 เคล็ดลับ: KPI ใช้เพื่อดูว่ารถหรือคนขับทำงานได้ดีแค่ไหน และควรปรับปรุงอะไรบ้าง ไม่ใช่เพื่อลงโทษ

📋 ข้อมูลหลัก (Master Data)

รถใช้งานหลัก

ความหมาย: รถที่ใช้งานเป็นหลักในงานประจำวัน

รถใช้งานสนับสนุน

ความหมาย: รถที่ใช้งานเสริมหรือสำรอง

เลขไมล์ (Mileage)

อธิบายง่ายๆ: เลขไมล์คือตัวเลขที่บอกว่ารถวิ่งไปแล้วกี่กิโลเมตร เหมือนตัวเลขบนหน้าปัดรถยนต์ที่บอกว่ารถวิ่งไปแล้วกี่กิโลเมตร

ความหมาย: ระยะทางที่รถวิ่งไปแล้ว (หน่วย: กิโลเมตร)

ใช้เพื่อ:

  • คำนวณอัตราสิ้นเปลือง: ดูว่ารถกินน้ำมันมากแค่ไหน
  • กำหนดรอบซ่อม: รู้ว่าควรซ่อมเมื่อไหร่ (เช่น ทุก 10,000 กม.)
  • ติดตามการใช้งาน: ดูว่ารถถูกใช้งานมากแค่ไหน
ตัวอย่าง:
• รถคันใหม่มีเลขไมล์ = 0 กม.
• หลังจากขับไป 1 เดือน มีเลขไมล์ = 1,500 กม.
• หลังจากขับไป 1 ปี มีเลขไมล์ = 15,000 กม.
• เมื่อถึง 20,000 กม. ควรเปลี่ยนน้ำมันเครื่อง
💡 เคล็ดลับ: ควรบันทึกเลขไมล์ทุกครั้งที่เติมน้ำมันหรือใช้รถ เพื่อให้ระบบคำนวณและเตือนได้ถูกต้อง

ภาษีรถยนต์

ความหมาย: ภาษีประจำปีที่ต้องชำระ

ระบบ: แจ้งเตือนล่วงหน้า 3 เดือนก่อนหมดอายุ

ประกันภัยภาคสมัครใจ

ความหมาย: ประกันภัยที่ซื้อเพิ่มเติมนอกเหนือจากประกันภาคบังคับ

ข้อมูล: เลขกรมธรรม์, เบี้ยประกัน, ทุนประกัน, วันเริ่ม-วันหมดอายุ

ระบบ: แจ้งเตือนล่วงหน้า 3 เดือนก่อนหมดอายุ

🔔 การแจ้งเตือน

Telegram Notification

อธิบายง่ายๆ: Telegram Notification คือ "ข้อความแจ้งเตือน" ที่ระบบส่งไปที่แอป Telegram ของคุณ เหมือนข้อความแจ้งเตือนจากแอปอื่นๆ แต่ส่งผ่าน Telegram

ความหมาย: การแจ้งเตือนผ่าน Telegram (แอปแชท)

เหตุการณ์ที่ระบบจะแจ้งเตือน:

  • สร้าง/แก้ไข/ยกเลิกคิว: เมื่อมีคนสร้าง แก้ไข หรือยกเลิกคิว
  • บันทึกการใช้รถ: เมื่อมีคนบันทึกการใช้รถ
  • ซ่อมบำรุง: เมื่อรถเข้าซ่อมหรือซ่อมเสร็จ
  • ภาษี/ประกันใกล้หมดอายุ: เมื่อภาษีหรือประกันใกล้หมดอายุ (3 เดือนก่อน)
  • น้ำมันผิดปกติ: เมื่อรถใช้น้ำมันผิดปกติ
ตัวอย่าง:
• เมื่อมีคนสร้างคิวใหม่ → คุณจะได้รับข้อความแจ้งเตือนว่า "มีคิวใหม่: ไปเชียงใหม่ วันที่ 20 ม.ค."
• เมื่อภาษีใกล้หมดอายุ → คุณจะได้รับข้อความแจ้งเตือนว่า "ภาษีรถ กข 1234 ใกล้หมดอายุใน 3 เดือน"
💡 เคล็ดลับ: การแจ้งเตือนช่วยให้คุณรู้ทันทีเมื่อมีอะไรเกิดขึ้น ไม่ต้องเข้าไปเช็คในระบบเอง

Flag ความผิดปกติ

ความหมาย: สัญลักษณ์เตือนเมื่อพบความผิดปกติ

ตัวอย่าง: ลืมบันทึกซ้ำบ่อย, น้ำมันผิดปกติ, ซ่อมบ่อย
