# คู่มือการขึ้นเว็บ วิธีใช้งาน และการทำคิวอาร์ — PPK DriveHub

**โปรเจกต์:** PPK DriveHub (ระบบจัดการยานพาหนะโรงเรียนพะเยาพิทยาคม 2569)

---

## ส่วนที่ 1 : วิธีนำโปรแกรมขึ้นเป็นเว็บ (Deploy)

ระบบมี 2 ส่วน: **Backend (Google Apps Script)** และ **Frontend (หน้าเว็บ)**

---

### ขั้นที่ 1 : เตรียมข้อมูลและโครงสร้าง

#### 1.1 สร้าง Google Spreadsheet และโฟลเดอร์ (ถ้ายังไม่มี)

- ใช้สคริปต์ Setup ตาม **SETUP_GUIDE.md**: เปิดไฟล์ `backend/Setup Folder Sheet Goole AppScript.gs` ใน GAS แล้วรันฟังก์ชัน `setupPPKDriveHub`  
- หรือสร้างเอง:
  - **Google Spreadsheet** ชื่อ `PPK-DriveHub-2569` พร้อม Sheet ตาม **SHEETS_SCHEMA.md** (USERS, USER_REQUESTS, QUEUE, CARS, DRIVERS, FUEL_LOG, REPAIR_LOG, CHECK_LOG ฯลฯ)
  - **โฟลเดอร์ Google Drive** ชื่อ `PPK-DriveHub-Uploads` และโฟลเดอร์ย่อย FUEL, REPAIR, CHECK, VEHICLES, DRIVERS ฯลฯ

#### 1.2 บันทึก ID ที่ต้องใช้

- **Spreadsheet ID** — จาก URL ของ Google Sheet:  
  `https://docs.google.com/spreadsheets/d/ **[SPREADSHEET_ID]** /edit`
- **ROOT_FOLDER_ID** — จาก URL ของโฟลเดอร์หลัก:  
  `https://drive.google.com/drive/folders/ **[ROOT_FOLDER_ID]**`

---

### ขั้นที่ 2 : Deploy Backend (Google Apps Script)

1. เปิด [Google Apps Script](https://script.google.com) → สร้างโปรเจกต์ใหม่  
2. **อัปโหลดไฟล์ทั้งหมด** จากโฟลเดอร์ `backend/` เข้าโปรเจกต์ (Code.gs, Config.gs, Utils.gs และไฟล์ .gs อื่นๆ ทั้งหมด)  
3. **แก้ไข Config.gs**:
   - `SPREADSHEET_ID` = ค่า Spreadsheet ID จากขั้น 1.2  
   - `SPREADSHEET_NAME` = `PPK-DriveHub-2569` (หรือชื่อที่ใช้)  
   - `ROOT_FOLDER_ID` = ค่าโฟลเดอร์หลักจากขั้น 1.2  
   - `QR_CODE_BASE_URL` = **URL หน้าเว็บ Frontend** ที่จะใช้ (เช่น `https://username.github.io/ppk-drivehub`) — ใช้สำหรับสร้างลิงก์ใน QR และลิงก์ในอีเมล  
4. **ตั้งค่า Script Properties** (ความลับ — ดู **DEPLOYMENT_SECRETS.md**):
   - Project Settings (ไอคอนฟันเฟือง) → Script Properties  
   - เพิ่ม `TELEGRAM_BOT_TOKEN` และ `TELEGRAM_CHAT_ID` (ถ้าใช้การแจ้งเตือน Telegram)  
5. **Deploy as Web App**:
   - Deploy → New deployment → เลือก type: **Web app**  
   - **Execute as:** Me  
   - **Who has access:** **Anyone** (สำคัญ เพื่อให้ Frontend เรียก API ได้)  
   - Deploy แล้ว **คัดลอก Web App URL** (รูปแบบ `https://script.google.com/macros/s/xxxxx/exec`)

---

### ขั้นที่ 3 : ตั้งค่าและ Deploy Frontend (หน้าเว็บ)

1. **แทนที่ URL ของ Backend ในทุกไฟล์ HTML**  
   ในโฟลเดอร์ `frontend/` ทุกไฟล์ที่มี `window.API_BASE_URL` ให้เปลี่ยนเป็น Web App URL ที่คัดลอกไว้ เช่น  
   `window.API_BASE_URL = 'https://script.google.com/macros/s/xxxxx/exec';`  

   วิธีแทนที่ทีละหลายไฟล์ (PowerShell ในโฟลเดอร์ frontend):
   ```powershell
   .\update-api-url.ps1 -ApiUrl "https://script.google.com/macros/s/xxxxx/exec"
   ```
   หรือใช้สคริปต์ `update-api-url.sh` บน Linux/Mac

2. **อัปโหลด Frontend ขึ้น hosting** อย่างใดอย่างหนึ่ง:
   - **GitHub Pages:** สร้าง repo แล้วอัปโหลดเนื้อหาใน `frontend/` ขึ้น branch `main` (หรือ `gh-pages`) เปิด GitHub Pages ที่ repo → จะได้ URL แบบ `https://username.github.io/repo-name/`
   - **Netlify / Vercel:** ลากโฟลเดอร์ `frontend/` ไปวาง หรือเชื่อม repo แล้วตั้ง root เป็น `frontend`
   - **เซิร์ฟเวอร์ของโรงเรียน:** อัปโหลดไฟล์ใน `frontend/` ไปที่โฟลเดอร์ที่เว็บเซิร์ฟเวอร์ชี้ (เช่น www, public_html)

3. **ตั้งค่า QR_CODE_BASE_URL ใน Config.gs** ให้ตรงกับ URL หน้าเว็บที่ได้ (เช่น `https://username.github.io/ppk-drivehub` โดยไม่มี slash ท้าย)

---

### ขั้นที่ 4 : สร้างผู้ใช้ Admin แรก

1. เปิด Google Spreadsheet → Sheet **USERS**  
2. เพิ่มแถวข้อมูลผู้ใช้ Admin (ตามโครงสร้างใน SHEETS_SCHEMA.md)  
   - คอลัมน์ `password_hash`: ใช้รหัสผ่านที่ต้องการ แล้วรันฟังก์ชัน `hashPassword('รหัสผ่านที่ตั้ง')` ใน GAS (รันจาก Editor) แล้วนำค่าที่ได้มาใส่  
   - อย่างน้อย: user_id, password_hash, full_name, role = `admin`, active = TRUE  

หรือใช้หน้า **สมัครใช้งาน** (register.html) แล้วให้ Admin คนแรก (ที่ใส่ใน Sheet เอง) เข้าไปอนุมัติคำขอสมัครจากนั้นใช้บัญชีที่อนุมัติแล้วเป็น Admin ได้

## ส่วนที่ 2 : วิธีใช้งาน

### ผู้ใช้ทั่วไป (หลังล็อกอิน)

1. **เข้าสู่ระบบ** — เปิดหน้า login (เช่น `https://your-site.github.io/ppk-drivehub/login.html`) กรอก user_id และรหัสผ่าน  
2. **เปลี่ยนรหัสผ่านครั้งแรก** — ถ้าระบบบังคับ จะพาไปหน้า change-password  
3. **ใช้งานตามสิทธิ์:**
   - **หน้าแรก (Dashboard)** — ดูภาพรวมสถิติ  
   - **คิวรถ** — ดู/สร้าง/แก้ไขคิว (ถ้ามีสิทธิ์ queue)  
   - **รถ / คนขับ** — ดู/เพิ่ม/แก้ไข (ถ้ามีสิทธิ์ vehicles, drivers)  
   - **น้ำมัน** — บันทึก/ดูน้ำมัน (fuel-record.html) (ถ้ามีสิทธิ์ fuel)  
   - **ซ่อมบำรุง** — บันทึก/ดูงานซ่อม (ถ้ามีสิทธิ์ repair)  
   - **รายงาน** — ดูรายงาน (ถ้ามีสิทธิ์ reports)  
   - **บันทึกการใช้งาน** — ดูประวัติการใช้งาน  
   - **ตั้งค่าส่วนตัว** — แก้ไขข้อมูลตัวเอง  
4. **ออกจากระบบ** — กด “ออกจากระบบ” ในเมนู

### Admin

1. **จัดการผู้ใช้** — ตั้งค่าระบบ (settings) → ดูคำขอสมัคร อนุมัติ/ปฏิเสธ และจัดการผู้ใช้ (แก้ไข, ระงับ, รีเซ็ตรหัสผ่าน)  
2. **จัดการรถ/คนขับ** — เพิ่ม/แก้ไขรถและคนขับ  
3. **คิวรถ** — สร้างคิว แก้ไข ยกเลิก ดู timeline  
4. **Audit Log / PDPA Log** — ดูจากเมนูตั้งค่าระบบ (ถ้ามีหน้า audit-log)  
5. **ตั้งค่าระบบ** — ค่าต่างๆ ตามที่ระบบออกแบบ

### การใช้ QR (ไม่ต้องล็อกอิน)

- **QR บันทึกใช้รถ** — สแกน QR ที่ติดรถ → เปิดหน้า qr-usage-record พร้อมรหัสรถ → กรอกเวลา/เลขไมล์ → บันทึก  
- **QR ตรวจเช็ครถ** — สแกน QR → เปิดหน้า qr-daily-check → กรอกผลตรวจเช็ค → บันทึก  
- **QR บันทึกน้ำมัน** — สแกน QR → เปิดหน้า qr-fuel-record → กรอกข้อมูลน้ำมัน → บันทึก  

รายละเอียดการทำ QR อยู่ในส่วนที่ 3

## ส่วนที่ 3 : วิธีการทำคิวอาร์ต่างๆ

**หลักการ:** คิวอาร์แยกเป็น **3 ส่วนอิสระ** — (1) บันทึกการใช้รถ (2) บันทึกการเติมน้ำมัน (3) บันทึกตรวจสภาพรถประจำวัน+แจ้งซ่อม  
หน้าเมนูที่เข้าดูแยกกันอิสระ (qr-usage-record / qr-fuel-record / qr-daily-check)  
**ไม่จำกัดสิทธิ์** — คนที่สแกน QR สามารถเข้าไปทำได้โดยไม่ต้องล็อกอิน; บนหน้า QR จะเห็นเฉพาะเมนู 3 ส่วนนี้ + ปุ่มเข้าสู่ระบบ

ระบบใช้ QR เป็น **ลิงก์ไปยังหน้าเว็บ** ที่มี parameter รหัสรถ (`car_id`) เพื่อให้รู้ว่าสแกนจากรถคันไหน

---

### 3.1 ค่า URL ที่ใช้กับ QR

- ใน **VehicleService.gs** เมื่อเพิ่มรถ ระบบสร้างค่า QR เป็น:  
  `CONFIG.QR_CODE_BASE_URL + '?car=' + carId`  
  ตัวอย่าง: ถ้า `QR_CODE_BASE_URL = 'https://yoursite.github.io/ppk-drivehub'` และ car_id = `abc-123` จะได้  
  `https://yoursite.github.io/ppk-drivehub?car=abc-123`

- หน้าเว็บรองรับการเปิดด้วย parameter:
  - **car** หรือ **qr** = รหัสรถ (car_id)  
  ตัวอย่าง: `qr-usage-record.html?car=abc-123`

- **ลิงก์ที่ใช้กับ QR แต่ละประเภท:**
  - บันทึกใช้รถ: `https://[เว็บคุณ]/qr-usage-record.html?car=[car_id]`
  - ตรวจเช็ครถ: `https://[เว็บคุณ]/qr-daily-check.html?car=[car_id]`
  - บันทึกน้ำมัน: `https://[เว็บคุณ]/qr-fuel-record.html?car=[car_id]`

---

### 3.2 ขั้นตอนสร้างและติด QR

#### วิธีที่ 1 : ใช้ค่าจากระบบ (หลังเพิ่มรถแล้ว)

1. ล็อกอินเป็น Admin หรือผู้มีสิทธิ์ vehicles  
2. เปิดหน้าจัดการรถ (vehicles) แล้วดูรายละเอียดรถที่ต้องการ  
3. ระบบจะแสดง **QR Code URL** (หรือค่า `qr_code` ของรถ) จาก Sheet CARS คอลัมน์ qr_code  
   - ค่าใน Sheet = `QR_CODE_BASE_URL + '?car=' + car_id`  
   - ตัวอย่าง: `https://yoursite.github.io/ppk-drivehub?car=xxxxx`  
4. ถ้าต้องการ QR แยกตามหน้าที่ใช้:
   - บันทึกใช้รถ: ใช้ URL `https://[เว็บคุณ]/qr-usage-record.html?car=[car_id]`
   - ตรวจเช็ค: `https://[เว็บคุณ]/qr-daily-check.html?car=[car_id]`
   - น้ำมัน: `https://[เว็บคุณ]/qr-fuel-record.html?car=[car_id]`

#### วิธีที่ 2 : สร้างภาพ QR Code จาก URL

1. เลือก URL ตามประเภท (ด้านบน) แล้วแทน `[car_id]` ด้วยรหัสรถจริงจาก Sheet CARS  
2. ใช้เว็บหรือแอปสร้าง QR จาก URL นั้น เช่น:
   - [QR Code Generator](https://www.qr-code-generator.com/) — ใส่ URL แล้วดาวน์โหลดรูป QR  
   - Google: ค้นหา "QR code generator" แล้ววาง URL  
   - แอปบนมือถือ: สร้าง QR จาก URL  
3. พิมพ์หรือแปะภาพ QR ที่รถ (ในตำแหน่งที่คนขับ/ผู้ใช้สแกนสะดวก)

#### วิธีที่ 3 : QR เดียวใช้ได้หลายหน้าที่ใช้

- ใช้ URL หลัก: `https://[เว็บคุณ]?car=[car_id]`  
  แล้วในหน้าแรก (index/dashboard) ให้มีปุ่มหรือลิงก์ไปยัง:
  - qr-usage-record.html?car=...
  - qr-daily-check.html?car=...
  - qr-fuel-record.html?car=...

หรือใช้ QR แยกอันต่อหน้าที่ใช้ (แนะนำเพราะเปิดตรงหน้าที่ต้องการ)

---

### 3.3 การใช้ QR แต่ละประเภท

| ประเภท | หน้าเว็บ | การทำงาน | ต้องล็อกอินหรือไม่ |
|--------|----------|----------|---------------------|
| บันทึกใช้รถ | qr-usage-record.html?car=xxx | สแกน → เปิดหน้ารหัสรถ xxx → กรอกเวลา ออก/กลับ ไมล์ → บันทึก (createUsageRecord / scanQRForUsageRecord) | ไม่ต้อง |
| ตรวจเช็ครถ | qr-daily-check.html?car=xxx | สแกน → เปิดหน้าเช็คสภาพรถ → กรอกผลตรวจ → บันทึก (createDailyCheck) | ไม่ต้อง |
| บันทึกน้ำมัน | qr-fuel-record.html?car=xxx | สแกน → เปิดหน้าน้ำมัน → กรอกจำนวน/ราคา ฯลฯ → บันทึก (createFuelLog) | ไม่ต้อง |

---

### 3.4 สรุปการตั้งค่าสำหรับ QR

1. **Config.gs** — ตั้ง `QR_CODE_BASE_URL` = URL หน้าเว็บ Frontend (ไม่มี slash ท้าย) เช่น  
   `https://username.github.io/ppk-drivehub`  
2. **สร้าง QR** — ใช้ URL รูปแบบ  
   `https://[เว็บคุณ]/qr-usage-record.html?car=[car_id]` (หรือ qr-daily-check / qr-fuel-record)  
   โดย `[car_id]` มาจาก Sheet CARS  
3. **ติด QR ที่รถ** — พิมพ์ภาพ QR แล้วแปะที่รถ  
4. **ผู้ใช้สแกน** — เปิดมือถือแล้วสแกน QR จะเข้าไปหน้าระบบพร้อมรหัสรถ ไม่ต้องล็อกอิน
