# การตั้งค่าและ Flow การบำรุงรักษารถ (ครั้งแรก และเมื่อแจ้งเตือนแล้วทำแล้ว)

## วัตถุประสงค์

- **ครั้งแรกที่ใช้โปรแกรม:** ตั้งค่าหรือให้ข้อมูลระบบอย่างไร (รถบางคันใช้งานมานาน ยังไม่เคยทำบางรายการ)
- **เมื่อแจ้งเตือนและมีการเปลี่ยนตามการแจ้งเตือนแล้ว:** ระบบจะเริ่มนับใหม่ได้อย่างไร
- **เน้น:** วางระบบได้ในโปรแกรม ยืดหยุ่น ใช้งานง่ายในระยะยาว รายละเอียดครบถ้วน

---

## 1. ครั้งแรกที่ใช้โปรแกรม — ตัวเลือกการตั้งค่า

### 1.1 เกณฑ์ร่วม (แอดมินตั้งครั้งเดียว)

- ไปที่ **ตั้งค่าระบบ (admin-settings)** → ส่วน **รายการที่ต้องแจ้งเตือน**
- ตั้ง **ระยะทาง (กม.)** และ **ระยะเวลา (เดือน)** ของแต่ละรายการ (เช่น น้ำมันเครื่อง 5,000 กม. / 6 เดือน)
- กด **บันทึกการตั้งค่า** → ระบบเก็บใน MASTER ใช้เป็นเกณฑ์ร่วมทุกคัน

### 1.2 ข้อมูล "ครั้งล่าสุดที่เปลี่ยน" ต่อคัน (เลือกลงได้ 3 แบบ)

รถแต่ละคันมีการเปลี่ยนในระยะที่ต่างกัน บางคันใช้งานมานานยังไม่เคยทำ จึงมี **3 วิธี** ให้เลือกตามความเหมาะสม:

| วิธี | เหมาะกับ | การทำ | ผลลัพธ์ |
|------|----------|--------|----------|
| **ก. รู้วันที่เปลี่ยนและเลขไมล์** | มีประวัติ/สมุดจด | บันทึก **last_km** และ **last_date** ของแต่ละรายการผ่าน "บันทึกการบำรุง" หรือฟอร์มซ่อม | ระบบนับ "ครบครั้งถัดไป" จาก last_km + เกณฑ์ กม. และ last_date + เกณฑ์ เดือน |
| **ข. ยังไม่เคยทำ / ไม่ทราบ** | คันเก่า ยังไม่เคยจด | ใช้ปุ่ม **"ใช้เลขไมล์และวันนี้เป็นจุดเริ่มต้น"** (ครั้งแรก) | ระบบตั้ง last_km = เลขไมล์ปัจจุบัน, last_date = วันนี้ ทุกรายการ → ครั้งถัดไปครบที่ (ไมล์ปัจจุบัน + เกณฑ์ กม.) และ (วันนี้ + เกณฑ์ เดือน) |
| **ค. ไม่กรอกอะไร** | ยังไม่พร้อมตั้ง | ไม่บันทึก last_km/last_date | ระบบใช้ fallback: ครบครั้งแรกที่ เกณฑ์ กม. จาก 0 หรือ เกณฑ์ เดือนจากวันจดทะเบียน (อาจแจ้งเตือนเร็วหรือเกิน due ขึ้นกับประวัติจริง) |

**แนะนำ:**  
- ถ้า **รู้ประวัติ** → ใช้ **ก.** บันทึก last_km / last_date จริง  
- ถ้า **ไม่ทราบ / ยังไม่เคยทำ** → ใช้ **ข.** "ใช้เลขไมล์และวันนี้เป็นจุดเริ่มต้น" เพื่อให้ระบบเริ่มนับจากวันนี้และไมล์ปัจจุบัน ไม่ย้อนไปใช้ 0 หรือวันจดทะเบียน

### 1.3 การกด "ใช้เลขไมล์และวันนี้เป็นจุดเริ่มต้น"

- เลือกรถคันที่ต้องการ → กดปุ่ม **"ตั้งค่าประวัติการบำรุงครั้งแรก – ใช้เลขไมล์และวันนี้"**
- ระบบจะบันทึกให้ทุกรายการ (น้ำมันเครื่อง, น้ำมันเกียร์, …) ว่า  
  **last_km = เลขไมล์ปัจจุบันของรถคันนี้** และ **last_date = วันนี้**
- หลังจากนั้น ระบบจะนับครั้งถัดไปว่า  
  ครบที่ **(last_km + เกณฑ์ กม.)** และ **(last_date + เกณฑ์ เดือน)**  
  เช่น น้ำมันเครื่อง 5,000 กม. / 6 เดือน → ครบครั้งถัดไปที่ ไมล์ปัจจุบัน+5,000 กม. และ วันนี้+6 เดือน

## 2. เมื่อแจ้งเตือนและมีการเปลี่ยนแล้ว — ระบบเริ่มนับใหม่ได้อย่างไร

หลัก: **เมื่อไหร่ก็ตามที่บันทึก "เปลี่ยนแล้ว" (last_km และ last_date ล่าสุด) ระบบจะเริ่มนับรอบใหม่จากจุดนั้น**

### 2.1 บันทึกผ่าน "บันทึกซ่อม" (ปิดงานซ่อม)

- ไปที่ **บันทึกซ่อมบำรุง** → เลือกรถ, วันที่ซ่อมเสร็จ, **เลขไมล์ตอนเข้าซ่อม**, รายการซ่อม (เช่น "เปลี่ยนถ่ายน้ำมันเครื่อง, เปลี่ยนไส้กรองอากาศ")
- กดบันทึก/ปิดงานซ่อม
- ระบบจะ **อัปเดต last_km และ last_date** ของรถคันนั้นให้อัตโนมัติ ตามรายการที่เลือก
- **ระบบเริ่มนับใหม่:** ครั้งถัดไป = last_km + เกณฑ์ กม. และ last_date + เกณฑ์ เดือน

### 2.2 บันทึกแบบ "ทำแล้ว" โดยไม่ผ่านใบซ่อมเต็ม

- เมื่อเห็น **แจ้งเตือน** (หรือไปที่หน้ารายละเอียดรถ / การบำรุง)
- กด **"บันทึกว่าทำแล้ว"** ของรายการนั้น
- กรอก (หรือใช้ค่าเริ่มต้น):
  - **เลขไมล์เมื่อเปลี่ยน** (default = เลขไมล์ปัจจุบันของรถ)
  - **วันที่เปลี่ยน** (default = วันนี้)
- กดบันทึก
- ระบบจะ **อัปเดต last_km และ last_date** ของรายการนั้นสำหรับรถคันนี้
- **ระบบเริ่มนับใหม่:** ครั้งถัดไป = last_km + เกณฑ์ กม. และ last_date + เกณฑ์ เดือน

### 2.3 สรุป Flow "เริ่มนับใหม่"

1. ระบบแจ้งเตือนเพราะใกล้ครบหรือเกินเกณฑ์ (เทียบจาก last_km, last_date เดิม)
2. ผู้ใช้ดำเนินการเปลี่ยน/ตรวจตามรายการนั้น
3. ผู้ใช้บันทึกหนึ่งในสองทาง: **(ก)** ปิดงานซ่อมใน "บันทึกซ่อม" หรือ **(ข)** กด "บันทึกว่าทำแล้ว" แล้วกรอกเลขไมล์และวันที่
4. ระบบอัปเดต last_km และ last_date ในชีต VEHICLE_MAINTENANCE
5. ระบบคำนวณรอบถัดไป = last_km + เกณฑ์ กม. และ last_date + เกณฑ์ เดือน → **นับใหม่จากจุดที่บันทึก**

---

## 3. ความยืดหยุ่นและรายละเอียดในระยะยาว

### 3.1 แก้ไขเมื่อไหร่ก็ได้

- **last_km / last_date** แก้ไขได้ตลอด (ผ่าน API หรือฟอร์ม "บันทึกการบำรุง" / "บันทึกว่าทำแล้ว")
- ถ้าบันทึกผิดหรือพบประวัติเก่า สามารถอัปเดต last_km และ last_date ใหม่ → ระบบจะนับจากค่าล่าสุดที่บันทึก

### 3.2 รถแต่ละคันแยกกัน

- แต่ละคันมีชุด last_km / last_date แยกใน **VEHICLE_MAINTENANCE** (car_id, item_key, last_km, last_date)
- รถคันเก่าที่เปลี่ยนนานแล้ว กับคันใหม่ที่เพิ่งเปลี่ยน จะได้แจ้งเตือนคนละเวลา ตามระยะจริงของแต่ละคัน

### 3.3 เกณฑ์ร่วม vs ข้อมูลต่อคัน

- **เกณฑ์ (ระยะ กม. / เดือน):** ตั้งครั้งเดียวใน **ตั้งค่าระบบ** ใช้ร่วมทุกคัน
- **จุดเริ่มนับแต่ละรอบ:** เก็บต่อคันใน **VEHICLE_MAINTENANCE** (last_km, last_date)
- ต้องการเปลี่ยนเกณฑ์ (เช่น น้ำมันเครื่องจาก 5,000 เป็น 6,000 กม.) → แก้ในตั้งค่าระบบอย่างเดียว ไม่ต้องแก้ทีละคัน

### 3.4 รายการที่ครบถ้วน

- รายการแจ้งเตือน (น้ำมันเครื่อง, น้ำมันเกียร์, ผ้าเบรก, ยาง, แบตเตอรี่ ฯลฯ) ตรงกับที่แอดมินตั้งใน "รายการที่ต้องแจ้งเตือน"
- แต่ละรายการมีได้ทั้ง **ระยะทาง (กม.)** และ **ระยะเวลา (เดือน)** ครบทั้งสองเงื่อนไข หรือใช้อย่างใดอย่างหนึ่ง (เช่น แบต 0 กม. / 36 เดือน)

## 4. สรุป Flow ทั้งระบบ (วางระบบในโปรแกรม)

```
[แอดมิน] ตั้งค่าระบบ → เกณฑ์ กม./เดือน ทุกรายการ (MASTER)
                ↓
[ครั้งแรกต่อคัน] เลือกหนึ่งในสาม:
   • บันทึก last_km / last_date จริง (ถ้ารู้)
   • กด "ใช้เลขไมล์และวันนี้เป็นจุดเริ่มต้น" (ถ้าไม่ทราบ/ยังไม่เคยทำ)
   • ไม่กรอก (ระบบใช้ fallback 0 / วันจดทะเบียน)
                ↓
[ระบบ] คำนวณครั้งถัดไป = last_km + เกณฑ์ กม. และ last_date + เกณฑ์ เดือน
                ↓
[ระบบ] แจ้งเตือนเมื่อใกล้ครบหรือเกิน (ตาม pm_advance)
                ↓
[ผู้ใช้] ทำการเปลี่ยน/ตรวจจริง
                ↓
[ผู้ใช้] บันทึกหนึ่งในสองทาง:
   • ปิดงานซ่อม (รายการซ่อมมีรายการนั้น) → อัปเดต last_km/last_date อัตโนมัติ
   • กด "บันทึกว่าทำแล้ว" → กรอกเลขไมล์และวันที่ → อัปเดต last_km/last_date
                ↓
[ระบบ] เริ่มนับรอบใหม่จาก last_km และ last_date ที่บันทึก
```

## 5. API / ฟีเจอร์ที่เกี่ยวข้อง (สำหรับพัฒนาต่อ)

- **getVehicleMaintenanceLast(carId)** — ดึง last_km / last_date ทุกรายการของรถคันนั้น
- **recordVehicleMaintenance(carId, updates)** — บันทึกหลายรายการ (เช่น หลังซ่อม)
- **setVehicleMaintenanceLast(carId, item_key, last_km, last_date, notes)** — บันทึกหนึ่งรายการ
- **initializeVehicleMaintenanceFromToday(carId)** — ตั้งค่าประวัติครั้งแรก: ใช้เลขไมล์ปัจจุบันและวันนี้เป็น last_km / last_date ทุกรายการ (สำหรับปุ่ม "ใช้เลขไมล์และวันนี้เป็นจุดเริ่มต้น")
- **getMaintenanceAlertsForVehicle(carId)** — คืนรายการที่ใกล้ครบหรือเกินเกณฑ์ของรถคันนั้น
